# 要件定義作成プロンプト

## 基本プロンプト
```
以下の概要に基づいて、詳細な要件定義書を作成してください。
要件定義書テンプレート（tsumiki-cursor/docs/requirements/requirements.md）に従って記載してください。

【プロジェクト概要】
{プロジェクトの概要をここに記載}

【重要な要件】
- {要件1}
- {要件2}
- {要件3}

【制約事項】
- {制約1}
- {制約2}

要件定義書には以下を含めてください：
1. 明確な機能要件（必須/オプション）
2. 非機能要件（パフォーマンス、セキュリティ、可用性など）
3. 技術的制約と前提条件
4. リスクと対策
5. 成功基準
```

## 詳細化プロンプト
```
この要件定義書の以下の点をより詳細に記載してください：
- {詳細化したい項目}

具体的には：
1. {具体的な要求1}
2. {具体的な要求2}
```

## レビュープロンプト
```
この要件定義書をレビューし、以下の観点で改善点を指摘してください：
1. 曖昧な表現や不明確な要件
2. 矛盾する要件
3. 不足している要件
4. 実現可能性の問題
5. セキュリティやパフォーマンスの懸念事項
```
