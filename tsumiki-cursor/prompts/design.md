# 設計文書作成プロンプト

## 基本プロンプト
```
要件定義書（tsumiki-cursor/docs/requirements/requirements.md）に基づいて、
詳細な設計文書を作成してください。
設計文書テンプレート（tsumiki-cursor/docs/design/design.md）に従って記載してください。

設計文書には以下を含めてください：
1. システム全体のアーキテクチャ図
2. 技術スタックの選定理由
3. データベース設計（ER図、テーブル定義）
4. API設計（エンドポイント、リクエスト/レスポンス）
5. 画面設計（画面遷移、レイアウト）
6. セキュリティ設計
7. エラーハンドリング方針
```

## データベース設計プロンプト
```
要件定義書に基づいて、データベース設計を行ってください：

1. ER図をMermaid形式で作成
2. 各テーブルの詳細定義（カラム名、型、制約）
3. インデックス設計
4. 正規化レベルとその理由
5. パフォーマンスを考慮した設計判断
```

## API設計プロンプト
```
要件定義書に基づいて、RESTful API設計を行ってください：

1. エンドポイント一覧（メソッド、パス、説明）
2. 各エンドポイントの詳細仕様
   - リクエストパラメータ/ボディ
   - レスポンス形式
   - エラーレスポンス
   - 認証要否
3. API設計のベストプラクティスに従う
```

## アーキテクチャレビュープロンプト
```
この設計文書のアーキテクチャをレビューし、以下の観点で評価してください：
1. スケーラビリティ
2. 保守性
3. テスタビリティ
4. セキュリティ
5. パフォーマンス
6. コスト効率

改善提案があれば具体的に示してください。
```
