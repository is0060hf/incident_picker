# タスク分割プロンプト

## 基本プロンプト
```
設計文書（tsumiki-cursor/docs/design/design.md）に基づいて、
実装タスクを適切に分割してください。
タスク管理テンプレート（tsumiki-cursor/docs/tasks/tasks.md）に従って記載してください。

タスク分割の基準：
1. 各タスクは1-2日で完了できる粒度
2. 依存関係を明確にする
3. テスト可能な単位で分割
4. 並行作業可能なタスクを識別

以下の観点でタスクを整理してください：
- フェーズ（基盤構築、機能実装、テスト、デプロイ）
- 優先度（必須、重要、あれば良い）
- 技術領域（フロントエンド、バックエンド、インフラ）
```

## 依存関係分析プロンプト
```
タスク一覧の依存関係を分析し、以下を作成してください：

1. タスク依存関係図（Mermaid形式）
2. クリティカルパス
3. 並行実行可能なタスクグループ
4. ボトルネックとなる可能性のあるタスク
```

## 見積もりプロンプト
```
各タスクの工数見積もりを行ってください：

1. 楽観的見積もり
2. 現実的見積もり
3. 悲観的見積もり
4. バッファを考慮した最終見積もり

見積もりの根拠も併せて記載してください。
```

## リスク評価プロンプト
```
タスク一覧を確認し、プロジェクトリスクを評価してください：

1. 技術的リスク
2. スケジュールリスク
3. リソースリスク
4. 外部依存リスク

各リスクに対する対策も提案してください。
```
